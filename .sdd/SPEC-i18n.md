# HermesDM Full i18n System — Implementation Plan

## Goal
Replace hardcoded Spanish UI strings in `telegram_handler.py` with a configurable language system, supporting English (`EN`) and Spanish (`ES`) via `CampaignSettings.language` (or `/language` command), with fallback to environment variable `DEFAULT_LANGUAGE`.

## Components

### 1. Translation Module (`bot/i18n.py`)
- Enum `Language` (reuse from `campaign_settings.Language`)
- Dict `TRANSLATIONS[Language][key]` with English and Spanish strings
- Function `get(key: str, language: Language) → str`
- Function `get_for_campaign(campaign_id: str, key: str) → str` (reads campaign settings)
- Safe fallback: if key missing, return English or key itself

### 2. Campaign Language Resolution
- `campaign_settings.py` already has `language: Language = Language.ES`
- Must load campaign state → `CampaignSettings.language`
- Default fallback: `DEFAULT_LANGUAGE` env var or `Language.EN`

### 3. Command `/language <es|en>`
- Change language for current campaign (or default for new campaigns)
- Persist in `CampaignSettings`, save to state.json

### 4. String Replacement in `telegram_handler.py`
- Identify all UI strings (help text, error messages, replies, prompts)
- Replace with `i18n.get("key", language)` calls
- Keep interpolation logic (`f"{variable}"`) inside translation strings

## Implementation Steps

**Phase 1 — Create i18n module**
1. Create `bot/i18n.py` with translation dict (ES, EN)
2. Add `get()` and `get_for_campaign()` functions
3. Add `/language` command handler that updates settings

**Phase 2 — String Extraction**
1. Scan `telegram_handler.py` for all Spanish strings
2. Map each unique string to a key (e.g., `no_campaign`)
3. Populate `TRANSLATIONS[Language.EN]` with English equivalents
4. Preserve Spanish in `TRANSLATIONS[Language.ES]`

**Phase 3 — Replacement**
1. Replace hardcoded strings with `i18n.get("key", language)` calls
2. Update `language` parameter usage in narrative generator (already done)
3. Ensure `CampaignSettings.language` is read per campaign

**Phase 4 — Testing**
1. Run bot, verify English responses
2. `/language es` → switch to Spanish
3. `/setup` flow works in both languages
4. Campaign persistence across restarts

## Risks & Mitigations
- **Missing key**: Fallback to English or raw key (no crash)
- **Interpolation errors**: Use `string.format(**context)` approach
- **Performance**: Translation dict in memory, no file I/O
- **Campaign migration**: Existing campaigns keep `language: ES` unless changed

## Post-Fix Features
- `/language en` → English bot replies
- `/language es` → Spanish bot replies
- New campaigns inherit `DEFAULT_LANGUAGE` env var
- Campaign-specific language persisted in `state.json`

## Out of Scope
- Multi-language within same chat (per-user language)
- Automatic language detection from user message
- Third languages (PT, etc.) beyond ES/EN
- Translation of narrative templates (already separate)
